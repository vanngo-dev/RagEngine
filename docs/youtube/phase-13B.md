# Phase 13B — Tauri Desktop UI

## Video Goal

Wrap the shared React UI from Slice 13A in a Tauri desktop shell without creating a second UI.

## What Changed

- Added Tauri configuration under `frontend/src-tauri/`.
- Added `npm run tauri:dev` and `npm run tauri:build` scripts.
- Configured Tauri dev mode to load the same Vite React app at `http://127.0.0.1:5173`.
- Configured production desktop builds to use the same `frontend/dist` generated by `npm run build`.
- Kept the backend as a manually started FastAPI service for this slice.

## Desktop Flow

```powershell
python -m uvicorn app.main:app --reload
cd frontend
npm run tauri:dev
```

If the backend is not running, the shared health panel shows the backend offline state.

## Validation

```powershell
python -m pytest -q
cd frontend
npm run build
```

Tauri validation attempted:

```powershell
cd frontend
npm run tauri:build
```

Result in this environment:

```text
Rust and Cargo are installed, and Cargo dependency resolution started successfully.
The build was blocked by Windows Application Control while executing a generated Cargo build script under frontend/src-tauri/target.
Error: An Application Control policy has blocked this file. (os error 4551)
```

## Known Limitations

- The desktop app does not bundle the FastAPI backend as a sidecar yet.
- A local Rust toolchain, platform-specific Tauri prerequisites, and an Application Control policy that permits Cargo build scripts are required for `tauri:dev` or `tauri:build`.
- The same React API client is used for web and desktop, defaulting to `http://127.0.0.1:8000`.
